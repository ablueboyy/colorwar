# ColorWar — 規格文件 (Spec v0.1)

## 1. 一句話定義

兩人線上即時對戰的「領地塗色 × 塔防」遊戲：玩家用隨時間累積的金錢購買各式砲台，發射顏色彈把地板染成自己的顏色、搶奪對方領地，遊戲結束時佔領面積較大者獲勝。

---

## 2. 核心玩法循環 (Core Loop)

```
累積金錢 ──▶ 購買/放置砲台 ──▶ 砲台自動發射染色彈 ──▶ 地板變色(佔領)
   ▲                                                          │
   └──────── 佔領面積越大，金錢產出越快(可選機制) ◀────────────┘
```

每一局：

1. 兩名玩家各從盤面一側出發，初始各佔一小塊己方顏色地板。
2. 金錢隨時間自動增加（基礎收入），佔領面積可額外加成收入。
3. 玩家把金錢花在砲台上：放置在自己領地內，砲台自動朝最近的敵方/中立地板發射染色彈。
4. 染色彈命中格子 → 該格變成發射方顏色。對方砲台可把它再染回去，形成拉鋸。
5. 計時結束（或某方達到佔領門檻）時結算，佔領格數多者勝。

---

## 3. 勝負條件

- **主要**：固定局長（預設 3 分鐘）。時間到，盤面上佔最多格子的玩家獲勝。
- **提前結束 (KO)**：任一方佔領 ≥ 80% 盤面，立即獲勝。
- **平手**：格數相同 → 以「累積染色次數」或「剩餘金錢」作為 tiebreaker（待定，見第 11 節）。

---

## 4. 盤面與座標系統

- **格狀盤面 (Grid)**：預設 `24 × 16` 格。每格有一個顏色狀態：
  - `P1`（玩家一色，預設藍）
  - `P2`（玩家二色，預設紅）
  - `NEUTRAL`（中立，灰）
- 砲台放在格子上，佔據 1 格（大型砲台可佔 2×2，待定）。
- 染色彈以連續座標飛行（非格對齊），命中時換算成所在格。

> 採格狀盤面的理由：佔領面積容易計算與顯示、放置邏輯單純、網路同步資料量小。

---

## 5. 砲台系統 (Towers)

每種砲台有：成本、射速、射程、單發染色範圍、彈速、**HP**、攻擊力（對砲台造成的傷害）。

**砲台可被摧毀（已決定）**：砲台有 HP，染色彈在染地板的同時，若飛過/命中敵方砲台也會對其造成傷害；HP 歸零則砲台被摧毀（不返還金錢）。這讓「保護自己砲台陣型 + 集火拆對方砲台」成為核心策略層。

| 砲台 | 成本 | HP | 行為 | 特性 |
|------|------|----|------|------|
| 基礎砲 (Basic) | 低 | 中 | 單發染 1 格 + 小傷害 | 便宜、通用 |
| 散射砲 (Spread) | 中 | 低 | 前方 3 格扇形染色 | 近距離快速擴張、脆 |
| 狙擊砲 (Sniper) | 中高 | 低 | 長射程、單格、高拆塔傷害 | 偷襲、專拆對方砲台 |
| 範圍砲 (Splash) | 高 | 中 | 命中點 3×3 範圍染色 | 大面積翻盤 |
| 護盾/加速器 (Support) | 中 | 高 | 不發射；提升周圍砲台射速或給鄰近砲台減傷 | 輔助、保護陣型 |

規則細節：

- **放置限制**：只能放在「自己顏色的格子」上。
- **腳下被染走**：若砲台腳下格子被對方染走，砲台**停止運作**（變灰、不再發射、仍可被攻擊）；若該格被己方重新染回則恢復運作。被染走時可手動售出，返還部分金錢。
- **傷害模型**：染色彈有 `colorDamage`（染地板）與 `towerDamage`（打砲台）兩個獨立屬性，狙擊砲 `towerDamage` 高、染色弱；範圍砲相反。
- **死亡回饋**：砲台被摧毀時播放爆炸特效、腳下格子變回中立（避免摧毀後仍留色造成混亂）。

---

## 6. 經濟系統 (Economy)

- 基礎收入：每秒 +N 金錢。
- **佔領加成（已決定採用）**：每佔領 1 格額外 +x 金錢/秒，鼓勵積極擴張。
  - **防滾雪球措施**（避免領先方提早鎖死勝局）：加成採「遞減曲線」而非線性 —— 例如前 30% 領地全額加成，超過部分加成減半；或設每秒加成上限 (cap)。實際曲線待 playtest 調。
- 砲台售出：返還部分成本（避免死局）；運作中砲台返還較多、被摧毀則不返還。
- 數值全部寫在 `config`，方便調平衡（見第 9 節）。

---

## 7. 線上對戰架構 (Networking)

採 **權威伺服器 (Authoritative Server)** 模型，避免作弊與不同步：

- 伺服器持有唯一遊戲狀態 (single source of truth)，跑固定 tick（預設 20 Hz）。
- 客戶端只送「意圖」指令：`PLACE_TOWER`、`SELL_TOWER`、`SELECT_TYPE`。
- 伺服器驗證（金錢夠不夠、格子是否屬於自己）後套用，廣播狀態。
- 客戶端負責渲染與插值，染色彈軌跡可在前端做視覺插值。

連線方式：

- 傳輸：**WebSocket**（JSON 訊息；之後若需要可換 binary）。
- 房間/配對：
  - MVP：建房 → 產生房間碼 → 對方輸入碼加入（2 人即開打）。
  - 之後可加：隨機快速配對。

訊息範例：

```jsonc
// Client → Server
{ "type": "PLACE_TOWER", "tower": "BASIC", "cell": { "x": 5, "y": 8 } }

// Server → Client (每 tick 或變更時)
{
  "type": "STATE",
  "tick": 1234,
  "board": "...",          // 壓縮後的格子顏色陣列
  "towers": [ /* ... */ ],
  "projectiles": [ /* ... */ ],
  "players": [ { "id": "P1", "money": 320, "cells": 142 }, ... ],
  "timeLeft": 96.4
}
```

## 8. 技術選型 (Tech Stack)

| 層 | 選擇 | 理由 |
|----|------|------|
| 前端渲染 | HTML5 Canvas 2D | 格狀盤面 + 大量彈幕，2D 足夠且簡單 |
| 前端語言 | TypeScript | 型別安全、與後端共用 model |
| 前端建置 | Vite | 快速 dev server / HMR |
| 後端 | Node.js + TypeScript | 與前端共用型別 (shared package) |
| 即時通訊 | `ws` (WebSocket) | 輕量、夠用 |
| 共用程式碼 | `packages/shared` | 遊戲常數、訊息型別、純函式邏輯共用 |
| 部署 | 單一 Node 服務同時供 static + WS | MVP 最簡 |

Monorepo 結構建議：

```
colorwar/
├─ packages/
│  ├─ shared/      # 型別、config、純邏輯 (tick 更新函式)
│  ├─ server/      # WebSocket 權威伺服器、房間管理
│  └─ client/      # Vite + Canvas 前端
├─ SPEC.md
└─ package.json    # workspaces
```

> 重點：把「遊戲狀態推進邏輯」(`stepGame(state, dt)`) 放在 `shared`，伺服器跑它做權威運算，客戶端也能用同一份做預測/重播，減少 bug。

---

## 9. 可調設定 (Config)

集中於 `shared/config.ts`，至少包含：

- 盤面尺寸、tick rate、局長
- 各砲台數值（成本/射速/射程/範圍/彈速）
- 經濟參數（基礎收入、佔領加成、售出返還比例）
- 勝利門檻（KO 百分比）

## 10. 開發里程碑 (Milestones)

- **M0 — 骨架**：monorepo、Vite client、Node WS server、能連線收發訊息。
- **M1 — 單機原型**：本地畫出格狀盤面、放砲台、發彈、染色、金錢累積（先不連線）。
- **M2 — 上線同步**：搬到權威伺服器，2 客戶端同房同步對戰。
- **M3 — 砲台多樣化**：實作全部砲台類型 + 經濟加成 + 售出。
- **M4 — 對局流程**：房間碼配對、計時、結算畫面、重新開始。
- **M5 — 打磨**：音效、特效、UI、平衡調整、斷線重連。

MVP = M0–M2（基礎砲台一種即可跑完整對戰）。

### 12.5 美術備註（像素風）

- **整體**：復古像素風 (pixel art)，砲台與彈幕用 sprite，地板用純色格子（像素色塊本身就很搭）。
- **解析度**：採低解析度 sprite（如砲台 16×16 或 32×32 px），Canvas 開 `imageSmoothingEnabled = false` 做硬邊放大，保留像素感。
- **資源**：MVP 可先用幾何色塊佔位 (placeholder)，M5 打磨階段再替換成正式像素 sprite / sprite sheet。
- **動畫**：砲台發射、爆炸、染色擴散用逐格 sprite 動畫；建議用 sprite sheet + 幀索引。
- **調色盤**：固定一組像素風 palette，P1 / P2 / 中立三主色 + 各自的暗部高光，維持視覺一致。
